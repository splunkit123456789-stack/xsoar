# SOAR 平台 P1 数据库设计

## 1. P1 数据库目标

P1 数据库支撑剧本审批流、审计日志、版本管理、APP 市场、告警通道、用户偏好等能力。

## 2. P1 新增 MySQL 表

| 表 | 用途 |
|---|---|
| `soar_approval` | 剧本审批记录 |
| `soar_approval_config` | 剧本审批流配置 |
| `soar_audit_log` | 审计日志 |
| `soar_workflow_version` | 剧本版本历史 |
| `soar_app_version` | APP 版本历史 |
| `soar_user_preference` | 用户偏好设置 |
| `soar_failure_alert_config` | 剧本失败告警配置 |

## 3. soar_approval_config 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `workflow_uuid` | `VARCHAR(64)` | 关联剧本 UUID |
| `approvers` | `JSON` | 审批人用户 ID 列表 |
| `approval_type` | `VARCHAR(32)` | single（任一通过）/ unanimous（会签） |
| `notify_channels` | `JSON` | 通知渠道配置 |
| `status` | `TINYINT` | 0=启用，1=禁用 |
| `create_time` | `DATETIME` | 创建时间 |
| `update_time` | `DATETIME` | 更新时间 |

索引：`uniq_workflow_uuid` (workflow_uuid)。

## 4. soar_approval 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `workflow_uuid` | `VARCHAR(64)` | 关联剧本 UUID |
| `workflow_name` | `VARCHAR(128)` | 剧本名称快照 |
| `trigger_user_id` | `INT` | 触发人 ID |
| `trigger_type` | `VARCHAR(32)` | manual/timer/webhook |
| `approver_id` | `INT` | 审批人 ID |
| `status` | `VARCHAR(32)` | pending/approved/rejected/timeout |
| `remarks` | `TEXT` | 审批意见 |
| `trigger_time` | `DATETIME` | 触发时间 |
| `approval_time` | `DATETIME` | 审批时间 |
| `timeout_at` | `DATETIME` | 审批超时时间 |
| `create_time` | `DATETIME` | 创建时间 |
| `update_time` | `DATETIME` | 更新时间 |

索引：
- `idx_workflow_uuid` (workflow_uuid, trigger_time)
- `idx_approver_status` (approver_id, status)

审批超时默认 24 小时，超时后自动标记为 `timeout`，剧本不执行。

## 5. soar_audit_log 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `BIGINT AUTO_INCREMENT` | 主键 |
| `user_id` | `INT` | 操作用户 ID |
| `user_account` | `VARCHAR(64)` | 操作用户账号快照 |
| `action_type` | `VARCHAR(64)` | 操作类型（login/create/update/delete/execute 等） |
| `resource_type` | `VARCHAR(64)` | 资源类型（workflow/user/role/app/variable 等） |
| `resource_id` | `VARCHAR(128)` | 资源 ID |
| `resource_name` | `VARCHAR(255)` | 资源名称快照 |
| `detail` | `JSON` | 操作详情（变更前后值等） |
| `ip` | `VARCHAR(64)` | 客户端 IP |
| `user_agent` | `VARCHAR(512)` | User-Agent |
| `status` | `VARCHAR(32)` | success/failed |
| `error_message` | `TEXT` | 失败原因 |
| `create_time` | `DATETIME` | 操作时间 |

索引：
- `idx_user_action_time` (user_id, action_type, create_time)
- `idx_resource_type_time` (resource_type, create_time)
- `idx_create_time` (create_time)

约束：
- 审计日志不可删除、不可修改。
- 保留期 180 天，超期归档（P2 实现）。
- `detail` 字段不记录明文密码、完整 Token。

## 6. soar_workflow_version 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `workflow_uuid` | `VARCHAR(64)` | 关联剧本 UUID |
| `version` | `INT` | 版本号，从 1 自增 |
| `name` | `VARCHAR(128)` | 剧本名称快照 |
| `flow_json` | `LONGTEXT` | 该版本的 flow_json |
| `flow_data` | `LONGTEXT` | 该版本的 flow_data |
| `controller_data` | `LONGTEXT` | 该版本的 controller_data |
| `local_var_data` | `LONGTEXT` | 该版本的 local_var_data |
| `remarks` | `VARCHAR(255)` | 备注 |
| `user_id` | `INT` | 创建者 ID |
| `create_time` | `DATETIME` | 创建时间 |

索引：`uniq_workflow_version` (workflow_uuid, version)。

约束：
- 每次剧本保存自动新增版本。
- 同一剧本最多保留 50 个版本，超过自动清理最早版本。
- 当前正在执行的版本不可删除。

## 10. P1 修改的 P0 表

### 10.1 soar_logs 表新增字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `replay_of` | `VARCHAR(64)` | 重跑来源 exec_id，非重跑为 NULL |
| `approval_id` | `INT` | 关联审批记录 ID，非审批触发为 NULL |

### 10.2 soar_app 表新增字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `current_version` | `VARCHAR(32)` | 当前版本号 |
| `package_path` | `VARCHAR(512)` | ZIP 包存储路径（导入的 APP） |
| `is_imported` | `TINYINT` | 是否为导入的 APP（0=否，1=是） |

## 11. 迁移约束

- P1 表结构变更通过 `migrations/mysql/000003_p1_tables.sql` 新增。
- 审计日志表使用 `BIGINT` 主键，预留足够空间。
- JSON 字段（`approvers`、`notify_channels`、`detail`、`channels`）必须有 schema 说明。
- APP ZIP 包存储路径建议为 `/data/app_packages/<app_code>/<version>.zip`。
