# SOAR 平台 P2 需求设计

## 1. 阶段目标

P2 面向规模化企业能力，重点建设 RBAC、多租户、分布式执行、SSO、报告导出、大屏可视化等高级特性。

P2 不改变 P1 的核心闭环，而是在权限治理、执行引擎扩展、企业身份集成方面增强。

## 2. P2 主要能力

| 模块 | P2 需求 |
|---|---|
| RBAC 细粒度权限 | 基于资源的细粒度权限控制（剧本查看/编辑/执行/删除独立授权） |
| 多租户/组织空间 | 支持多组织/多团队隔离，每组织独立数据空间 |
| 分布式执行引擎 | 基于 RPyC 实现多 Worker 节点调度，剧本可分配到指定 Worker |
| SSO 单点登录 | 支持 LDAP/OAuth2/SAML/CAS 等企业 SSO 协议接入 |
| 剧本执行报告导出 | 执行完成后自动生成 PDF/HTML 执行报告 |
| 大屏可视化 | 提供执行态势大屏，实时展示全平台剧本执行情况 |
| APP 开发 SDK 与脚手架 | 提供标准化的 APP 开发 SDK、CLI 脚手架、模板项目 |
| 剧本调试模式 | 支持单节点调试、断点调试、变量实时查看 |
| 资产/CMDB 集成 | 提供与常见 CMDB 集成的 APP，支持按资产标签批量操作 |
| SIEM 集成与告警双向同步 | 与 Splunk/ES/QRadar 等 SIEM 集成，支持告警自动拉取并触发剧本 |
| 性能监控与限流 | 监控平台自身 QPS/延迟/资源占用，对高频剧本做限流与排队 |
| 备份与恢复 | 支持剧本/变量/APP/配置的定时自动备份与一键恢复 |
| API 开放平台 | 提供完善的 OpenAPI 文档（Swagger）、API Key 管理、调用限流、调用日志 |
| 国际化 i18n | 支持中文/英文等多语言切换 |

## 3. RBAC 细粒度权限 P2 要求

### 3.1 权限模型

- RBAC1：用户 → 角色 → 权限点。
- 资源级权限：剧本、APP、变量、分类等资源支持独立的 `read/write/execute/delete` 权限点。
- 资源 scope：限制用户可访问的资源范围（按 UUID 列表或通配符）。

### 3.2 内置角色

| 角色 | 说明 |
|---|---|
| `super_admin` | 超级管理员，拥有全部权限，不可删除 |
| `org_admin` | 组织管理员，管理本组织内所有资源 |
| `workflow_editor` | 剧本编辑者，可创建/编辑/执行剧本 |
| `workflow_viewer` | 剧本查看者，只读权限 |
| `auditor` | 审计员，只可查看审计日志 |

### 3.3 资源级权限点

| 资源 | 权限点 |
|---|---|
| 剧本 | `workflow:read`、`workflow:write`、`workflow:execute`、`workflow:delete` |
| APP | `app:read`、`app:write`、`app:delete` |
| 变量 | `variable:read`、`variable:write`、`variable:delete` |
| 用户 | `user:read`、`user:write`、`user:delete` |
| 角色 | `role:read`、`role:write`、`role:delete` |
| 审计日志 | `audit:read` |
| 系统设置 | `system:read`、`system:write` |

### 3.4 资源 scope

```json
{
  "workflow_scopes": {
    "read": ["wf-uuid-1", "wf-uuid-2"],
    "write": ["wf-uuid-1"],
    "execute": ["*"],
    "delete": []
  }
}
```

- `*` 表示全部资源。
- 未授权资源不出现在用户视图中。
- 绕过前端直接请求 API 时，后端必须执行权限和 scope 校验。

## 4. 多租户/组织空间 P2 要求

### 4.1 组织模型

- 顶层组织（Tenant）：独立的命名空间和数据隔离单元。
- 子组织（Organization）：Tenant 下的子团队。
- 用户可属于多个组织，每次登录选择当前组织。

### 4.2 数据隔离

- 所有业务表新增 `tenant_id` 字段。
- 所有查询自动注入 `tenant_id = <current_tenant>` 条件。
- 跨租户访问数据返回 `FORBIDDEN`。

### 4.3 资源配额

- 每租户可配置资源配额：最大剧本数、最大 APP 数、最大执行并发数。
- 超出配额返回 `QUOTA_EXCEEDED`。

## 5. 分布式执行引擎 P2 要求

### 5.1 Worker 节点模型

- Worker 是独立进程，通过 RPyC 连接到 Master。
- Worker 注册时上报：hostname、IP、CPU 核数、内存、可用 APP 列表。
- Master 维护 Worker 心跳（5s 一次），心跳超时 30s 后标记为 offline。

### 5.2 任务分配策略

| 策略 | 说明 |
|---|---|
| `random` | 随机选择 Worker（默认） |
| `round_robin` | 轮询 |
| `least_loaded` | 选择负载最低的 Worker |
| `specified` | 指定 Worker（剧本配置中指定） |
| `app_aware` | 根据 APP 依赖选择有对应 APP 的 Worker |

### 5.3 故障切换

- Worker 执行任务时心跳超时，Master 标记任务为 `worker_lost`。
- 配置了 `retry_on_worker_lost = true` 的剧本自动重新调度到其他 Worker。
- 重试次数限制：3 次。

## 6. SSO 单点登录 P2 要求

### 6.1 支持的协议

| 协议 | 说明 |
|---|---|
| LDAP | 支持 LDAP v3，配置 server_url/bind_dn/bind_password/base_dn |
| OAuth2 | 支持 Authorization Code Flow，配置 client_id/client_secret/authorize_url/token_url/userinfo_url |
| SAML 2.0 | 支持 SP-initiated SSO，配置 idp_metadata_url |
| CAS | 支持 CAS 2.0/3.0，配置 server_url |

### 6.2 用户映射

- SSO 用户首次登录时自动创建 SOAR 用户（账号 = SSO 用户名）。
- 可配置默认角色和组织。
- SSO 用户与本地用户使用相同的 Token 机制。

### 6.3 多 SSO 源

- 支持同时配置多个 SSO 源（如 LDAP + OAuth2）。
- 登录页展示已配置的 SSO 入口按钮。

## 7. 剧本执行报告导出 P2 要求

- 执行完成后自动生成执行报告（PDF + HTML）。
- 报告内容：剧本元信息、执行摘要、每节点详情（输入/输出/耗时/状态）、错误堆栈、变量快照。
- 报告存储路径：`/data/reports/<exec_id>.{pdf,html}`。
- 报告保留期：90 天，超期自动清理。
- 执行历史详情页提供"下载报告"按钮。

## 8. 大屏可视化 P2 要求

- 大屏入口：导航栏 → "大屏"。
- 大屏内容：
  - 顶部：剧本总数、今日执行数、成功率、当前并发数。
  - 中间：最近 24 小时执行趋势折线图、按剧本分类的执行次数排行。
  - 底部：最近 10 条执行记录滚动展示、当前异常告警列表。
- 大屏自适应分辨率（1920x1080 / 2560x1440 / 4K）。
- 数据刷新：5s 一次（通过 WebSocket 推送）。
- 大屏可投屏展示（全屏模式）。

## 9. APP 开发 SDK 与脚手架 P2 要求

### 9.1 APP 开发 SDK

- Python SDK：`soar-app-sdk`，提供：
  - APP 元信息装饰器 `@soar_app`、`@soar_action`。
  - 参数校验、日志记录、错误处理的基类。
  - 本地调试入口（`python main.py --debug`）。
  - 自动生成 app.json 的 CLI 工具。

### 9.2 脚手架

```bash
soar init my-app --template=basic
cd my-app
soar run --debug
soar package  # 打包成 ZIP
soar upload --url=http://soar.example.com  # 上传到 SOAR 实例
```

### 9.3 APP 模板

| 模板 | 说明 |
|---|---|
| `basic` | 最简单的 helloworld 模板 |
| `http_request` | HTTP 请求 APP 模板 |
| `database` | 数据库操作 APP 模板 |
| `notification` | 消息通知 APP 模板 |

## 10. 剧本调试模式 P2 要求

### 10.1 单节点调试

- 剧本编辑器中，每个节点右上角提供"调试"按钮。
- 点击后弹出调试面板，可手动输入参数，执行单节点。
- 调试输出展示在面板下方。

### 10.2 断点调试

- 节点可设置断点（节点左侧边栏点击切换）。
- 执行到断点节点时暂停，展示当前所有变量值。
- 用户可：
  - 继续执行（F5）
  - 单步执行（F10）
  - 查看变量
  - 修改变量
  - 终止执行

### 10.3 变量实时查看

- 调试模式下，右侧面板展示所有局部变量、全局变量、节点输出。
- 变量值支持展开查看（JSON 树形展示）。

## 11. 资产/CMDB 集成 P2 要求

### 11.1 内置 CMDB APP

- 提供 `cmdb` APP，支持以下动作：
  - `query_assets`：按标签/IP/主机名查询资产列表。
  - `get_asset_detail`：查询单个资产详情。
  - `update_asset_tag`：更新资产标签。

### 11.2 CMDB 配置

- 系统设置中配置 CMDB 连接信息：
  - API URL
  - API Key
  - 资产标签字段映射

### 11.3 剧本中按资产批量操作

- 剧本中调用 `cmdb.query_assets` 获取资产列表。
- 通过 `foreach` 循环节点对每个资产执行操作。

## 12. SIEM 集成与告警双向同步 P2 要求

### 12.1 支持的 SIEM

| SIEM | 集成方式 |
|---|---|
| Splunk | 通过 Splunk REST API 拉取告警 |
| Elastic SIEM | 通过 Elasticsearch API 拉取告警 |
| QRadar | 通过 QRadar REST API 拉取告警 |
| 开源 SIEM | 通过 Webhook 推送告警 |

### 12.2 告警拉取

- 配置 SIEM 连接信息和告警查询条件。
- 系统定时（可配置间隔）拉取新告警。
- 告警触发对应剧本（通过告警类型 → 剧本映射）。

### 12.3 告警处置回写

- 剧本执行完成后，将处置结果回写到 SIEM（关闭告警、添加备注）。
- 回写失败时记录错误，不阻塞剧本执行。

## 13. 性能监控与限流 P2 要求

### 13.1 平台自身监控

- 监控指标：QPS、平均响应时间、P95/P99 响应时间、错误率、CPU/内存占用。
- 监控数据通过 Prometheus 暴露（`/metrics` 端点）。
- 提供 Grafana Dashboard 模板。

### 13.2 限流与排队

- 单剧本并发执行数限制（默认 10）。
- 全局并发执行数限制（默认 100）。
- 超出限制的执行请求进入排队队列。
- 排队队列长度限制（默认 1000），超限返回 `QUEUE_FULL`。

## 14. 备份与恢复 P2 要求

### 14.1 自动备份

- 每日凌晨 2:00 自动备份。
- 备份内容：所有 MySQL 表 + APP 包 + 配置文件。
- 备份存储路径：`/data/backups/<date>/`。
- 备份保留期：30 天，超期自动清理。

### 14.2 手动备份

- 系统设置中提供"立即备份"按钮。
- 备份完成后生成下载链接。

### 14.3 恢复

- 系统设置中提供"恢复"功能。
- 选择备份文件（本地上传或从备份列表选择）。
- 恢复前提示用户确认，警告将覆盖现有数据。
- 恢复完成后自动重启服务。

## 15. API 开放平台 P2 要求

### 15.1 OpenAPI 文档

- 通过 Swagger UI 暴露 OpenAPI 3.0 文档（`/api/docs`）。
- 文档自动从 API 代码生成。
- 提供在线调试能力。

### 15.2 API Key 管理

- 用户可创建多个 API Key（命名、权限范围、过期时间）。
- API Key 可禁用、删除。
- API Key 调用日志记录在审计日志中。

### 15.3 调用限流

- 每个 API Key 默认限流：100 次/分钟。
- 超出限流返回 `429 Too Many Requests`，响应头包含 `X-RateLimit-Limit`、`X-RateLimit-Remaining`、`X-RateLimit-Reset`。

## 16. 国际化 i18n P2 要求

- 支持语言：`zh-CN`（默认）、`en-US`。
- 用户偏好中可切换语言。
- 所有 UI 文案通过 i18n key 引用，不硬编码中文。
- 后端错误消息支持多语言（通过 `Accept-Language` 头判断）。
- 时间格式根据语言切换：`zh-CN` 使用 `YYYY-MM-DD HH:MM:SS`，`en-US` 使用 `MM/DD/YYYY HH:MM:SS`。

## 17. P2 不包含

- AI 智能剧本生成（进入 P3）。
- AI 异常根因分析（进入 P3）。
- APP 商店（进入 P3）。
- 低代码可视化大屏搭建（进入 P3）。
- ChatOps 集成（进入 P3）。
- Kubernetes Operator 化（进入 P3）。
- 边缘轻量化部署（进入 P3）。
- 区块链审计存证（进入 P3）。
- 可观测性集成（OpenTelemetry，进入 P3）。
- 多云混合编排（进入 P3）。
