# SOAR 平台 P2 数据库设计

## 1. P2 数据库目标

P2 数据库支撑 RBAC 细粒度权限、多租户、分布式执行、SSO、报告导出、大屏可视化、备份恢复、API 开放平台、国际化能力。

## 2. P2 新增 MySQL 表

| 表 | 用途 |
|---|---|
| `soar_permission` | 权限点注册表 |
| `soar_role_permission` | 角色-权限点绑定 |
| `soar_role_resource_scope` | 角色资源 scope |
| `soar_tenant` | 租户表 |
| `soar_tenant_user` | 租户-用户绑定 |
| `soar_tenant_quota` | 租户配额 |
| `soar_worker` | Worker 节点注册表 |
| `soar_sso_provider` | SSO Provider 配置 |
| `soar_siem_provider` | SIEM Provider 配置 |
| `soar_siem_alert_mapping` | 告警 → 剧本映射 |
| `soar_execution_report` | 执行报告元信息 |
| `soar_api_key` | API Key 管理 |
| `soar_backup` | 备份记录 |

## 3. RBAC 相关表

### 3.1 soar_permission 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `code` | `VARCHAR(64)` | 权限点编码，唯一（如 `workflow:read`） |
| `name` | `VARCHAR(128)` | 权限点名称 |
| `description` | `VARCHAR(255)` | 描述 |
| `resource_type` | `VARCHAR(64)` | 资源类型（workflow/app/variable 等） |
| `action` | `VARCHAR(32)` | 动作（read/write/execute/delete） |

### 3.2 soar_role_permission 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `role_id` | `INT` | 角色 ID |
| `permission_code` | `VARCHAR(64)` | 权限点编码 |

索引：`uniq_role_permission` (role_id, permission_code)。

### 3.3 soar_role_resource_scope 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `role_id` | `INT` | 角色 ID |
| `resource_type` | `VARCHAR(64)` | 资源类型 |
| `action` | `VARCHAR(32)` | 动作（read/write/execute/delete） |
| `resource_pattern` | `VARCHAR(255)` | 资源 ID 或通配符（`*`、`wf-uuid-1`） |

索引：`idx_role_resource` (role_id, resource_type)。

## 12. 迁移约束

- P2 表结构变更通过 `migrations/mysql/000004_p2_tables.sql` 新增。
- 所有业务表新增 `tenant_id` 字段，需通过 `ALTER TABLE ... ADD COLUMN ... DEFAULT 1` 迁移。
- 迁移完成后，需执行数据回填脚本，确保所有现有数据归属默认租户。
- SSO/SIEM 配置中的敏感字段（如密码）必须加密存储，加密密钥通过环境变量 `SOAR_ENCRYPTION_KEY` 注入。
