# SOAR 平台 P3 需求设计

## 1. 阶段目标

P3 面向生态与前瞻能力，引入 AI 智能化能力，构建 SOAR 生态，扩展边缘场景。

P3 不改变 P2 的核心闭环，而是在 AI 能力、生态建设、云原生部署、可观测性方面增强。

## 2. P3 主要能力

| 模块 | P3 需求 |
|---|---|
| AI 智能剧本生成 | 接入大模型（GPT/GLM/通义），通过自然语言描述自动生成可执行的剧本 JSON |
| AI 异常根因分析 | 剧本执行失败时由 AI 分析错误日志，给出根因与修复建议 |
| APP 商店（云市场） | 官方运营云端 APP 商店，支持一键安装、版本升级、评分评价 |
| 低代码可视化大屏搭建 | 提供可视化大屏搭建工具，剧本执行数据实时投屏 |
| ChatOps 集成 | 与 Slack/钉钉/飞书/企微等 IM 集成，支持在 IM 中通过指令触发剧本 |
| 剧本模板库 | 提供官方剧本模板库，用户基于模板快速创建剧本 |
| Kubernetes Operator 化 | 提供 K8s CRD 与 Operator，支持在 K8s 集群中以声明式方式部署和管理剧本 |
| 边缘轻量化部署 | 提供资源占用极低的边缘 Agent，支持在边缘节点执行剧本片段 |
| 区块链审计存证 | 关键审计日志上链存证，确保审计记录不可篡改、可独立验证 |
| 可观测性集成 | 集成 OpenTelemetry/Prometheus/Grafana，剧本执行 Metrics/Trace/Log 全量上报 |
| 多云混合编排 | 支持阿里云/腾讯云/AWS/华为云/私有云的混合编排，统一凭证管理 |

## 3. AI 智能剧本生成 P3 要求

### 3.1 大模型接入

- 支持的大模型：
  - OpenAI GPT-4 / GPT-3.5
  - Anthropic Claude 2 / Claude Instant
  - 阿里通义千问
  - 智谱 GLM-4 / GLM-5
  - 百度文心一言
- 用户在系统设置中配置大模型 API Key 和模型选择。

### 3.2 自然语言生成剧本

用户在剧本编辑器中输入自然语言描述，例如：

```
每天凌晨 2 点检查服务器磁盘空间，
如果超过 80% 就发送钉钉告警给运维群
```

系统调用大模型，生成对应的剧本 JSON：

```json
{
  "name": "磁盘空间检查",
  "timer": {"type": "cron", "cron": "0 2 * * *"},
  "nodes": [
    {"app": "linux", "action": "check_disk_space", "args": {}}
  ],
  "branches": [
    {"condition": "${node-1.usage_percent} > 80", "action": "send_dingding_alert"}
  ]
}
```

### 3.3 生成结果验证

- 生成结果后，系统自动校验剧本 JSON 的合法性。
- 校验通过：用户可预览剧本，确认后保存。
- 校验失败：系统提示错误，用户可重新生成或手动修改。

### 3.4 上下文学习

- 系统记录用户的历史剧本，作为大模型的上下文。
- 生成结果更符合用户的剧本风格和命名习惯。

## 4. AI 异常根因分析 P3 要求

### 4.1 自动分析

- 剧本执行失败时，系统自动调用 AI 分析错误日志。
- AI 分析以下信息：
  - 失败节点的输入参数
  - 失败节点的错误堆栈
  - 上游节点的输出
  - 剧本的整体配置

### 4.2 分析结果展示

- 在执行历史详情页的失败节点处展示 AI 分析结果。
- 分析结果包含：
  - Top 3 可能的失败原因
  - 每个原因的修复建议
  - 相关文档/示例链接

### 4.3 一键修复

- 对于常见的修复操作（如修改参数、增加重试），AI 提供"一键修复"按钮。
- 用户点击后，系统自动应用修复，并重新执行剧本。

## 5. APP 商店（云市场）P3 要求

### 5.1 商店架构

- 官方运营云端 APP 商店：`https://store.soar.io`
- 商店后端：APP 元信息数据库、用户评价数据库、下载统计数据库。
- 商店前端：APP 浏览/搜索/详情/评分/下载页面。

### 5.2 商店功能

| 功能 | 说明 |
|---|---|
| 浏览 APP | 按分类浏览（消息通知/数据库/安全工具/运维/集成等） |
| 搜索 APP | 按名称/标签/描述搜索 |
| APP 详情 | 展示 APP 元信息、版本历史、使用文档、截图、评分评价 |
| 一键安装 | 点击"安装"按钮，APP 自动下载并导入到本地 SOAR 实例 |
| 版本升级 | 商店展示 APP 的最新版本，用户可一键升级 |
| 评分评价 | 安装后可对 APP 评分（1-5 星）和评价 |
| 上架 APP | 开发者可提交 APP 到商店，经审核后上架 |

### 5.3 商店集成

- 在 SOAR 控制台的"APP 管理"页面，提供"浏览商店"入口。
- 商店页面嵌入 SOAR 控制台，用户无需跳转到外部网站。
- 安装 APP 时，自动调用本地 SOAR 实例的 APP 导入 API。

## 6. 低代码可视化大屏搭建 P3 要求

### 6.1 大屏编辑器

- 提供可视化大屏编辑器，用户拖拽组件搭建大屏。
- 组件类型：
  - 卡片组件：展示单个指标值
  - 折线图组件：展示时间序列数据
  - 柱状图组件：展示分类数据
  - 饼图组件：展示占比数据
  - 表格组件：展示列表数据
  - 滚动列表组件：展示实时滚动数据
  - 仪表盘组件：展示进度/百分比
- 组件可配置数据源（API 端点、刷新频率、过滤条件）。
- 大屏可保存为模板，复用到其他场景。

### 6.2 大屏预览与发布

- 编辑器提供"预览"功能，展示大屏效果。
- 用户确认后，点击"发布"，大屏变为只读模式，可通过 URL 访问。
- 已发布的大屏可继续编辑，编辑后重新发布。

### 6.3 大屏分享

- 发布后的大屏有独立的 URL，可通过链接分享。
- 支持嵌入到其他系统（iframe）。
- 支持 token 鉴权，控制访问权限。

## 7. ChatOps 集成 P3 要求

### 7.1 支持的 IM 平台

| 平台 | 集成方式 |
|---|---|
| Slack | Slack Bot API |
| 钉钉 | 钉钉机器人 + outgoing 回调 |
| 飞书 | 飞书机器人 + 事件订阅 |
| 企业微信 | 企微机器人 + 回调 |
| Telegram | Telegram Bot API |

### 7.2 IM 指令

用户在 IM 群中通过指令与 SOAR 交互：

| 指令 | 说明 | 示例 |
|---|---|---|
| `/soar list` | 列出剧本 | `/soar list --type=安全运营` |
| `/soar exec <uuid>` | 触发剧本执行 | `/soar exec wf-uuid-1` |
| `/soar status <exec_id>` | 查询执行状态 | `/soar status exec-001` |
| `/soar logs <exec_id>` | 查询执行日志 | `/soar logs exec-001` |
| `/soar help` | 展示帮助 | `/soar help` |

### 7.3 IM 通知

- 剧本执行完成/失败时，可配置自动推送到 IM 群。
- 推送内容：剧本名称、执行 ID、状态、关键输出。
- 推送格式：卡片消息（含执行详情链接）。

### 7.4 IM 交互式卡片

- 推送的执行结果卡片支持按钮交互。
- 例如：执行失败时，卡片展示"重试"、"查看详情"按钮。
- 用户点击按钮，触发对应操作。

## 8. 剧本模板库 P3 要求

### 8.1 官方模板库

- 官方维护剧本模板库，覆盖常见安全/运维场景：
  - 应急响应模板（钓鱼邮件处置、勒索软件处置、数据泄露处置）
  - 漏洞管理模板（漏洞扫描、漏洞验证、补丁部署）
  - 合规检查模板（等保检查、GDPR 合规、PCI-DSS 合规）
  - 运维自动化模板（磁盘清理、日志归档、备份验证）
  - DevOps 模板（CI/CD 触发、部署验证、回滚）

### 8.2 模板管理

- 模板库页面展示所有模板，按分类筛选。
- 模板详情页展示：剧本结构、所需 APP、参数说明、使用文档。
- 用户点击"使用模板"，进入剧本编辑器，预填充模板内容。
- 用户可基于模板创建自定义剧本。

### 8.3 模板贡献

- 用户可将自研剧本贡献为模板。
- 贡献的模板经官方审核后上架。
- 贡献者获得社区积分和展示。

## 9. Kubernetes Operator 化 P3 要求

### 9.1 CRD 定义

SOAR Operator 提供以下 CRD：

```yaml
apiVersion: soar.io/v1alpha1
kind: Workflow
metadata:
  name: phishing-response
  namespace: soar-system
spec:
  uuid: wf-uuid-1
  flow_json: |
    {"nodes": [...], "edges": [...]}
  timer:
    type: cron
    cron: "0 2 * * *"
  replicas: 1
```

### 9.2 Operator 功能

- 监听 `Workflow` CRD 变更，自动同步到 SOAR 后端。
- 监听 `Timer` CRD，自动创建/更新定时任务。
- 提供 `Execution` CRD，用户可通过 kubectl 触发执行：

```bash
kubectl create -f - <<EOF
apiVersion: soar.io/v1alpha1
kind: Execution
metadata:
  name: exec-001
  namespace: soar-system
spec:
  workflow_uuid: wf-uuid-1
  trigger_type: manual
EOF
```

### 9.3 部署方式

- SOAR 后端部署为 K8s Deployment。
- Operator 部署为 K8s Deployment，使用 Helm Chart。
- 提供 `helm install soar soar/soar` 一键部署。

## 10. 边缘轻量化部署 P3 要求

### 10.1 边缘 Agent

- 边缘 Agent 是 SOAR 的轻量化版本，部署在边缘节点。
- 资源占用：
  - 内存：< 50MB
  - CPU：< 5%（空载时）
  - 磁盘：< 100MB
- 启动时间：< 5s。
- 不依赖 MySQL/Redis，使用 SQLite + 内存缓存。

### 10.2 边缘剧本执行

- Master 将剧本片段下发到边缘 Agent。
- 边缘 Agent 本地执行剧本片段。
- 执行结果异步上报到 Master。

### 10.3 离线缓存

- 边缘 Agent 与 Master 网络中断时，剧本片段继续执行。
- 执行结果缓存到本地 SQLite，网络恢复后上报。
- 缓存容量：1000 条执行记录。
- 超过缓存容量时，覆盖最旧的记录。

## 11. 区块链审计存证 P3 要求

### 11.1 区块链选择

- 公链：以太坊、Polygon、币安智能链（BSC）
- 联盟链：Hyperledger Fabric、FISCO BCOS
- 国内合规：蚂蚁链、腾讯云区块链、长安链

### 11.2 存证策略

- 关键审计日志（登录、剧本执行、配置变更、权限变更）上链存证。
- 存证内容：审计日志的哈希值、时间戳、操作用户。
- 不上链敏感数据（如密码、Token）。

### 11.3 验证机制

- 提供"验证审计日志"功能，用户可验证某条审计日志是否被篡改。
- 验证流程：
  1. 从 MySQL 读取审计日志，计算哈希值。
  2. 从区块链读取对应哈希值。
  3. 比对哈希值，一致则验证通过。

### 11.4 区块链浏览器

- 提供 Web 界面的区块链浏览器，查看存证记录。
- 支持按时间范围、用户、操作类型筛选存证记录。
- 每条存证记录可查看交易哈希、区块高度、确认数。

## 12. 可观测性集成 P3 要求

### 12.1 OpenTelemetry 集成

- SOAR 后端集成 OpenTelemetry SDK。
- 自动埋点：
  - HTTP 请求：方法、路径、状态码、耗时
  - 数据库查询：SQL 语句、耗时
  - 剧本执行：剧本 UUID、节点 ID、耗时、状态
- Trace 数据通过 OTLP 协议上报到 Jaeger/Tempo。
- Metric 数据通过 OTLP 协议上报到 Prometheus。
- Log 数据通过 OTLP 协议上报到 Loki。

### 12.2 Grafana Dashboard

- 提供官方 Grafana Dashboard 模板，包含：
  - 平台概览：QPS、延迟、错误率、活跃用户数
  - 剧本执行：执行次数、成功率、P95 延迟、热门剧本
  - 系统资源：CPU、内存、磁盘、网络
  - 数据库：查询次数、慢查询、连接数
- 用户可通过导入 JSON 模板快速创建 Dashboard。

### 12.3 剧本执行 Trace

- 每次剧本执行生成一个 Trace，包含所有节点的 Span。
- Span 属性：
  - `workflow.uuid`：剧本 UUID
  - `workflow.name`：剧本名称
  - `node.id`：节点 ID
  - `node.app`：APP 编码
  - `node.action`：节点动作
  - `node.status`：节点状态
  - `node.duration_ms`：节点耗时
- Trace 可在 Jaeger 中查看，展示剧本执行的完整调用链。

## 13. 多云混合编排 P3 要求

### 13.1 支持的云平台

| 云平台 | 集成方式 |
|---|---|
| 阿里云 | 阿里云 SDK（ECS、RDS、SLB、OSS、安全产品） |
| 腾讯云 | 腾讯云 SDK（CVM、CDB、CLB、COS、安全产品） |
| AWS | AWS SDK（EC2、RDS、ELB、S3、安全产品） |
| 华为云 | 华为云 SDK（ECS、RDS、ELB、OBS、安全产品） |
| 私有云 | OpenStack SDK / vSphere SDK |

### 13.2 凭证管理

- 系统提供"云凭证管理"页面，统一管理各云平台的 Access Key / Secret Key。
- 凭证加密存储，仅在执行时解密。
- 凭证可关联到组织/租户，实现多租户凭证隔离。

### 13.3 多云 APP

- 提供统一的多云 APP，屏蔽各云平台的 API 差异。
- 多云 APP 支持的操作：
  - 虚拟机生命周期：创建/启动/停止/重启/销毁
  - 数据库生命周期：创建/备份/恢复/销毁
  - 负载均衡：添加/移除后端服务器
  - 安全产品：WAF 规则更新、安全组规则更新
- 多云 APP 通过参数指定目标云平台。

### 13.4 跨云编排示例

```yaml
workflow:
  name: 跨云扩容 + WAF 更新
  nodes:
    - app: alicloud
      action: scale_out_ecs
      args: {cluster: prod-cluster, count: 5}
    - app: tencentcloud
      action: update_waf_rule
      args: {rule_id: waf-001, src_ip: ${node-1.new_instances_ip}}
```

## 14. P3 不包含

- 完整替代 Splunk SOAR / Demisto / Phantom。
- 完整 SIEM 告警分析能力（SOAR 是 SOAR 平台，不是 SIEM）。
- 完整 DevOps 流水线（Jenkins/GitLab CI 替代品）。
- 商业 SOAR 产品的所有功能。
- 上述功能的具体实现细节（仅提供需求与设计方向）。
