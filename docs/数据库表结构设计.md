# SOAR 平台 数据库表结构设计

本文档汇总 SOAR 平台 所有阶段的数据库表结构。详细 DDL 按阶段拆分，本文件维护总览和设计口径。

## 1. 存储总览

| 存储引擎 | 用途 | 数据量预估 |
|---|---|---|
| MySQL 8.0+ | 配置元数据、用户、角色、权限、审计日志 | 中小（< 100GB） |
| Redis 6+ | 会话缓存、Token、定时调度状态、执行计数 | 小（< 1GB） |
| 文件系统 | APP 包存储、备份文件、执行报告 | 中（< 100GB） |

## 2. 表继承关系

参数 `xdp` 项目规范，`W5_P{0,1,2}_数据库设计.md` 维护各阶段新增表，本文件维护 4 个阶段合并后的完整表结构。

### 3.1 MVP 表

#### soar_users

```sql
CREATE TABLE `soar_users` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `account` VARCHAR(64) NOT NULL COMMENT '登录账号',
  `passwd` VARCHAR(128) NOT NULL COMMENT 'MD5 加密密码',
  `nick_name` VARCHAR(64) DEFAULT '' COMMENT '昵称',
  `email` VARCHAR(128) DEFAULT '' COMMENT '邮箱',
  `avatar` VARCHAR(255) DEFAULT '' COMMENT '头像路径',
  `status` TINYINT DEFAULT 0 COMMENT '0=启用，1=禁用',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY `uk_account` (`account`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户账号';
```

## 4. 迁移约束

- 数据库结构变更必须通过 `migrations/mysql/` 新增迁移文件，按顺序命名：
  - `000001_core_metadata.sql` — MVP 核心表
  - `000002_p0_tables.sql` — P0 新增表
  - `000003_p1_tables.sql` — P1 新增表
  - `000004_p2_tables.sql` — P2 新增表
- 迁移脚本必须幂等，支持多次执行。
- 新增字段必须通过 `ALTER TABLE` 迁移，不直接修改线上表。
- 所有表必须包含 `create_time` 和 `update_time`。
- 配置类表必须包含 `deleted_at` 软删除字段。
- 生产环境迁移前必须在测试环境验证。
