# SOAR 平台 P2 人工验收

## 1. 验收目标

验证 P2 的 RBAC 细粒度权限、多租户/组织空间、分布式执行引擎、SSO 单点登录、剧本执行报告导出、大屏可视化、APP 开发 SDK 与脚手架、剧本调试模式、资产/CMDB 集成、SIEM 集成与告警双向同步、性能监控与限流、备份与恢复、API 开放平台、国际化 i18n 真实生效。

## 3. RBAC 细粒度权限验收

### 3.1 创建资源级权限角色

1. 创建角色 `剧本只读`，授予权限点 `workflow:read`，资源 scope `workflow.read = ["wf-uuid-1"]`。
2. 创建用户 `viewer1`，绑定角色 `剧本只读`。
3. 使用 `viewer1` 登录。
4. **通过标准**：用户只能看到 `wf-uuid-1` 剧本，其他剧本不可见。

### 3.2 资源 scope 通配符验收

1. 创建角色 `剧本全读`，授予 `workflow:read`，资源 scope `workflow.read = ["*"]`。
2. **通过标准**：该角色用户能看到所有剧本。

### 3.3 后端越权拦截验收

1. 使用 `viewer1` 调用剧本编辑 API（`PUT /api/v1/soar/workflows/wf-uuid-1`）。
2. **通过标准**：返回 `403 FORBIDDEN`，错误码 `FORBIDDEN`，包含 `required_permission: workflow:write`。

3. 使用 `viewer1` 调用剧本删除 API。
4. **通过标准**：返回 `403 FORBIDDEN`。

### 4.4 跨租户访问拦截

### 4.5 配额超限

## 5. 分布式执行引擎验收

### 5.1 注册 Worker

1. 在另一台机器上启动 SOAR Worker 进程：
   ```bash
   python run_worker.py --master=10.0.0.1:9999 --hostname=worker-2
   ```
2. 在 Master 管理界面查看 Worker 列表。
3. **通过标准**：Worker 列表展示 `worker-2`，状态为 `online`。

### 5.2 任务分配策略验收

1. 启动 3 个 Worker 节点。
2. 配置剧本 `worker 测试`，`worker_assignment.strategy = specified`，`worker_id = worker-2`。
3. 执行剧本。
4. **通过标准**：执行日志中展示 `worker_id = worker-2`，该任务在 worker-2 上执行。

5. 修改剧本，`strategy = round_robin`。
6. 连续执行 6 次。
7. **通过标准**：3 个 Worker 各执行 2 次（轮询）。

### 5.3 故障切换验收

1. 剧本配置 `retry_on_worker_lost = true`。
2. 执行剧本，任务分配到 worker-2。
3. 在任务执行过程中，强制 kill worker-2 进程。
4. **通过标准**：
   - Master 检测到 worker-2 心跳超时（30s）。
   - 任务自动重新调度到其他 Worker。
   - 剧本最终执行成功。

## 6. SSO 单点登录验收

### 6.1 配置 LDAP SSO

1. 进入"系统设置 → SSO 配置"。
2. 点击"新增 SSO Provider"。
3. 配置：
   - 名称：`企业 LDAP`
   - 类型：`ldap`
   - Server URL：`ldap://ldap.example.com:389`
   - Bind DN：`cn=admin,dc=example,dc=com`
   - Bind Password：`<password>`
   - Base DN：`ou=users,dc=example,dc=com`
   - Username Attribute：`uid`
   - Default Role：`剧本编辑者`
   - Default Tenant：`租户 A`
4. 点击"测试连通性"。
5. **通过标准**：测试成功，提示"LDAP 连接正常"。

### 6.2 SSO 登录

1. 退出当前账号，回到登录页。
2. 点击"企业 LDAP 登录"按钮。
3. 输入 LDAP 账号密码。
4. **通过标准**：
   - 登录成功，进入控制台。
   - 用户列表中新增该 SSO 用户（账号 = LDAP uid）。
   - 用户默认角色为"剧本编辑者"，默认租户为"租户 A"。

### 6.3 SSO 用户二次登录

1. 上面的 SSO 用户退出登录。
2. 再次通过 LDAP 登录。
3. **通过标准**：登录成功，不重复创建用户。

## 13. 性能监控与限流验收

### 13.1 Prometheus 指标

```bash

```

**通过标准**：返回 Prometheus 格式的指标，包含：
- `soar_http_requests_total{method, path, status}`

- `soar_workflow_executions_total{status}`
- `soar_workflow_execution_duration_seconds`
- `soar_active_concurrent_executions`

### 13.2 剧本执行限流

1. 修改系统限流配置：单剧本最大并发数 `2`，全局最大并发数 `5`。
2. 对同一剧本发起 5 次并发执行请求。
3. **通过标准**：
   - 前 2 次立即执行。
   - 第 3-5 次进入排队队列。
   - 队列中的任务按 FIFO 顺序执行。

### 13.3 排队队列超限

1. 修改排队队列长度限制为 `2`。
2. 对同一剧本发起 10 次并发执行请求。
3. **通过标准**：
   - 前 2 次立即执行。
   - 第 3-4 次进入排队队列。
   - 第 5 次起返回 `503 QUEUE_FULL`。

