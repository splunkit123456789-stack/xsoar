# SOAR 平台 P0 需求设计

## 1. 阶段目标

P0 是上线阻断项。没有 P0 能力，就无法面向企业团队日常运营 SOAR 平台。

P0 在 MVP 基础上补齐：定时触发器、Webhook 触发器、用户与角色管理、RBAC 权限控制、剧本分类、全局变量、仪表盘、APP 分组、执行结果查询接口、CORS 跨域、WebSocket 实时推送。

## 2. P0 必须完成项

| 模块 | P0 要求 |
|---|---|
| 登录 | Token 过期机制、登出清理本地 Token、记住我 |
| 用户管理 | 用户 CRUD、角色管理、用户-角色绑定 |
| RBAC 权限控制 | 资源级权限（剧本/APP/变量读写执行）、后端 API 鉴权 |
| APP 管理 | APP 分组管理、APP 启用/禁用、APP 按分组筛选 |
| 剧本管理 | 剧本 CRUD、剧本分类管理、剧本按分类筛选 |
| 剧本触发 | 定时触发器（cron/interval/date）、Webhook 触发器、手动触发 |
| 变量管理 | 全局变量 CRUD、剧本中引用全局变量 |
| 仪表盘 | 剧本总数、APP 总数、执行次数、成功率/失败率、最近执行趋势 |
| 执行查询 | 通过 API 查询剧本执行状态、成功/失败数、执行日志 |
| 实时推送 | 剧本执行时通过 WebSocket 推送节点状态变化 |
| 跨域支持 | 后端启用 CORS，允许前端跨域调用 |

## 3. 用户与角色管理 + RBAC 权限控制 P0 要求

### 3.1 用户与角色管理

- 用户列表展示账号、昵称、邮箱、角色、状态、创建时间。
- 新增用户必填账号、密码、昵称；账号在未删除用户内唯一。
- 编辑用户可修改昵称、邮箱、密码、头像、角色。
- 删除用户：不允许删除自己；删除后清理 Token 和角色绑定。
- 角色列表展示角色名、描述、用户数、状态。
- 内置管理员角色（`super_admin`）不可删除，拥有全部权限。

### 3.2 RBAC 权限模型

- 权限模型：用户 → 角色 → 权限点。一个用户可绑定一个角色，角色关联多个权限点。
- 后端授权是唯一安全边界，前端菜单隐藏只做体验优化。
- 每个受保护 API 必须经过统一鉴权中间件校验，未授权返回 `403 FORBIDDEN`。

### 3.3 权限点定义

| 资源 | 权限点 | 说明 |
|---|---|---|
| 剧本 | `workflow:read` | 查看剧本列表和详情 |
| 剧本 | `workflow:write` | 创建和编辑剧本 |
| 剧本 | `workflow:execute` | 执行剧本 |
| 剧本 | `workflow:delete` | 删除剧本 |
| APP | `app:read` | 查看 APP 列表和详情 |
| APP | `app:manage` | 启用/禁用 APP |
| 变量 | `variable:read` | 查看全局变量 |
| 变量 | `variable:write` | 创建和编辑全局变量 |
| 变量 | `variable:delete` | 删除全局变量 |
| 定时器 | `timer:read` | 查看定时任务 |
| 定时器 | `timer:manage` | 创建/暂停/恢复/删除定时任务 |
| 用户管理 | `user:read` | 查看用户列表 |
| 用户管理 | `user:manage` | 创建/编辑/删除用户 |
| 角色管理 | `role:read` | 查看角色列表 |
| 角色管理 | `role:manage` | 创建/编辑/删除角色 |
| 审计日志 | `audit:read` | 查看审计日志 |
| 系统设置 | `system:read` | 查看系统设置 |
| 系统设置 | `system:write` | 修改系统设置 |

### 3.4 内置角色

| 角色 | 拥有的权限 | 说明 |
|---|---|---|
| `super_admin` | 全部权限 | 超级管理员，不可删除 |
| `workflow_editor` | `workflow:read/write/execute/delete`, `app:read`, `variable:read/write/delete`, `timer:read/manage` | 剧本编辑者 |
| `workflow_executor` | `workflow:read/execute`, `app:read`, `variable:read`, `timer:read` | 仅可执行剧本 |
| `viewer` | `workflow:read`, `app:read`, `variable:read` | 只读用户 |

### 3.5 权限校验规则

- 菜单权限：根据当前用户角色权限点，控制左侧导航菜单项的可见性。
- API 权限：每个受保护 API 标注所需权限点，鉴权中间件自动校验。
- 越权访问：用户直接请求未授权 API 时，返回 `403 FORBIDDEN`，错误码包含 `required_permission` 字段。
- 超级管理员：`super_admin` 角色跳过所有权限校验。
- P0 暂不实现资源级 scope（如"仅能查看指定剧本"），所有权限点为全局生效。

## 4. 定时触发器 P0 要求

- 触发方式：date（单次）、interval（间隔）、cron（表达式）。
- date 触发：配置 `run_date`，到点执行一次。
- interval 触发：配置 `interval_type`（seconds/minutes/hours/days/weeks）和 `time`，按间隔循环执行。
- cron 触发：配置标准 5 段 cron 表达式（分 时 日 月 周）。
- 支持配置 `start_date`、`end_date`、`jitter`（抖动秒数）。
- 定时任务可暂停、恢复、删除。
- 定时任务状态在剧本详情页展示。
- Cron 表达式校验：必须是 5 段，每段取值范围合法。

## 5. Webhook 触发器 P0 要求

- 剧本启用 Webhook 触发后，系统生成唯一的 `webhook_app`（UUID）。
- 外部系统通过以下接口触发剧本：
  ```
  POST /api/v1/soar/webhook
  {
    "key": "<api_key>",
    "uuid": "<webhook_app>",
    "data": <任意 JSON>
  }
  ```
- `api_key` 通过系统设置管理，存入 `soar_setting`。
- Webhook 触发的剧本可通过 `data` 字段接收外部数据。
- Webhook 触发的剧本在执行日志中 `trigger_type = webhook`。
- 剧本未启用 Webhook 触发时，调用 webhook 接口返回 `WEBHOOK_NOT_ENABLED`。

## 6. 全局变量管理 P0 要求

- 全局变量列表展示变量名、值、描述、更新时间。
- 新增变量必填变量名（小写字母+下划线，唯一）和值。
- 变量值支持 string/number/boolean/json 四种类型。
- 剧本中通过 `${var.var_name}` 语法引用全局变量。
- 修改全局变量后，下次执行生效。
- 删除全局变量前校验是否被剧本引用，被引用则返回 `VAR_IN_USE`。

## 7. 仪表盘 P0 要求

- 顶部卡片展示：剧本总数、APP 总数、今日执行次数、成功率。
- 中间折线图展示最近 7 天执行趋势（按天聚合）。
- 底部饼图展示最近 30 天执行状态分布（success/failed/timeout）。
- 数据实时刷新（页面刷新或定时 5 分钟轮询）。
- P0 仪表盘数据通过实时 SQL 聚合查询，不要求物化视图。

## 8. APP 分组管理 P0 要求

- APP 分组列表展示分组名、APP 数量、排序、状态。
- 新增分组必填分组名（未删除分组内唯一）。
- APP 编辑时可选择所属分组。
- APP 列表页支持按分组筛选。
- 内置分组"未分类"不可删除，所有未指定分组的 APP 归入此分组。

## 9. 剧本分类管理 P0 要求

- 剧本分类列表展示分类名、剧本数量、状态。
- 新增分类必填分类名（未删除分类内唯一）。
- 剧本编辑时可选择所属分类。
- 剧本列表页支持按分类筛选。
- 内置分类"默认分类"不可删除。

## 10. P0 不包含

- 多租户、组织隔离。
- 资源级 scope（如"仅能查看指定剧本"），P0 权限点为全局生效。
- 剧本审批流、审计日志。
- 条件分支、循环、并行节点。
- 剧本版本管理、导入导出。
- 剧本执行报告导出。
- 分布式执行引擎。
- SSO 单点登录。
