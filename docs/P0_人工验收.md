## 4. 定时触发器验收

### 4.1 配置 Interval 定时触发器

1. 创建剧本 `定时测试剧本`，编排 helloworld 节点。
2. 进入剧本详情页，点击"定时触发器"配置。
3. 选择"间隔触发"，间隔类型 `minutes`，间隔数值 `1`，抖动 `0`。
4. 保存配置。
5. **通过标准**：定时任务列表展示新建任务，状态为"启用"。

### 4.2 验证定时执行

1. 等待 1 分钟。
2. 查看执行日志。
3. **通过标准**：执行日志中有 `trigger_type = timer` 的记录，节点状态为 success。

### 4.3 配置 Cron 定时触发器

1. 进入剧本详情页，配置 cron 触发器：`0 2 * * *`（每天凌晨 2 点）。
2. **通过标准**：保存成功，定时任务列表展示 cron 表达式。

### 4.4 Cron 表达式校验

1. 配置 cron 触发器：`invalid cron`。
2. **通过标准**：返回错误 `TIMER_CRON_INVALID`。

## 5. Webhook 触发器验收

### 5.1 启用 Webhook 触发

1. 创建剧本 `Webhook 测试剧本`，编排 helloworld 节点，参数 `name = ${data.alert_id}`。
2. 进入剧本详情页，点击"Webhook 触发器"。
3. 点击"启用 Webhook"。
4. **通过标准**：生成 `webhook_app` UUID，展示在页面上。

### 5.2 通过 Webhook 触发剧本

```bash
curl -X POST http://127.0.0.1:8888/api/v1/soar/webhook \
  -H "Content-Type: application/json" \
  -d '{
    "key": "<api_key>",
    "uuid": "<webhook_app UUID>",
    "data": {"alert_id": "alert-001", "src_ip": "10.0.1.8"}
  }'
```

**通过标准**：
- 返回 `status: success`，包含 `exec_id`。
- 执行日志中有 `trigger_type = webhook` 的记录。
- 节点输出包含 `Hello alert-001`。

### 5.3 Webhook 错误验收

1. 使用错误的 API Key 调用 webhook 接口。
2. **通过标准**：返回错误 `WEBHOOK_KEY_INVALID`。

3. 使用不存在的 webhook UUID 调用 webhook 接口。
4. **通过标准**：返回错误 `WEBHOOK_UUID_NOT_FOUND`。

## 10. 执行查询 API 验收

```bash
# 通过 API Key 触发剧本执行
curl -X POST http://127.0.0.1:8888/api/v1/soar/api/exec \
  -H "Content-Type: application/json" \
  -d '{
    "key": "<api_key>",
    "uuid": "<workflow_uuid>",
    "data": {}
  }'

# 查询执行状态
curl -X GET "http://127.0.0.1:8888/api/v1/soar/api/exec_status?exec_id=<exec_id>&key=<api_key>"

# 查询执行日志
curl -X GET "http://127.0.0.1:8888/api/v1/soar/api/exec_logs?exec_id=<exec_id>&key=<api_key>"
```

**通过标准**：
- `/api/exec` 返回 `exec_id`。
- `/api/exec_status` 返回执行状态。
- `/api/exec_logs` 返回执行日志详情。
