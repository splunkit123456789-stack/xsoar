# SOAR 平台 P0 数据库设计

## 1. P0 数据库目标

P0 数据库设计支撑用户角色、RBAC 权限控制、剧本分类、APP 分组、定时触发器、Webhook 触发器、全局变量、仪表盘统计。

## 2. P0 新增 MySQL 表

| 表 | 用途 |
|---|---|
| `soar_role` | 角色定义 |
| `soar_user_role` | 用户-角色绑定 |
| `soar_permission` | 权限点注册表 |
| `soar_role_permission` | 角色-权限点绑定 |
| `soar_workflow_type` | 剧本分类 |
| `soar_app_group` | APP 分组 |
| `soar_timer` | 定时触发器配置 |
| `soar_variable` | 全局变量 |

## 3. soar_role 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `name` | `VARCHAR(64)` | 角色名，唯一 |
| `description` | `VARCHAR(255)` | 描述 |
| `is_builtin` | `TINYINT` | 是否内置角色（0=否，1=是） |
| `status` | `TINYINT` | 0=启用，1=禁用 |
| `create_time` | `DATETIME` | 创建时间 |
| `update_time` | `DATETIME` | 更新时间 |

约束：
- 内置管理员角色 `is_builtin = 1`，不可删除。
- 角色通过 `soar_role_permission` 表关联权限点。

## 4. soar_user_role 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `user_id` | `INT` | 用户 ID |
| `role_id` | `INT` | 角色 ID |
| `create_time` | `DATETIME` | 创建时间 |

索引：`uniq_user_role` (user_id, role_id)。

## 5. soar_permission 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `code` | `VARCHAR(64)` | 权限点编码，唯一（如 `workflow:read`） |
| `name` | `VARCHAR(128)` | 权限点名称 |
| `description` | `VARCHAR(255)` | 描述 |
| `resource_type` | `VARCHAR(64)` | 资源类型（workflow/app/variable/timer/user/role 等） |
| `action` | `VARCHAR(32)` | 动作（read/write/execute/delete/manage） |

约束：
- 权限点编码格式为 `资源:动作`，例如 `workflow:read`。
- 系统初始化时预置 18 个权限点，对应 P0 需求设计中的权限点定义表。

## 6. soar_role_permission 表

| 字段 | 类型 | 说明 |
|---|---|---|

索引：`uniq_role_permission` (role_id, permission_code)。

## 9. soar_timer 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INT AUTO_INCREMENT` | 主键 |
| `timer_uuid` | `VARCHAR(64)` | 定时任务 UUID，唯一 |
| `workflow_uuid` | `VARCHAR(64)` | 关联剧本 UUID |
| `type` | `VARCHAR(32)` | date/interval/cron |
| `cron` | `VARCHAR(128)` | Cron 表达式（type=cron 时） |
| `interval_type` | `VARCHAR(32)` | seconds/minutes/hours/days/weeks |
| `time` | `INT` | 间隔数值 |
| `run_date` | `DATETIME` | 单次执行时间（type=date 时） |
| `start_date` | `DATETIME` | 生效起始时间 |
| `end_date` | `DATETIME` | 生效结束时间 |
| `jitter` | `INT` | 抖动秒数 |
| `status` | `TINYINT` | 0=启用，1=暂停，2=已删除 |
| `create_time` | `DATETIME` | 创建时间 |
| `update_time` | `DATETIME` | 更新时间 |

索引：`idx_workflow_uuid` (workflow_uuid)。

## 12. Redis 使用扩展

| Key 模式 | 用途 |
|---|---|
| `token:{token}` | Token → user_id 映射，TTL 24h |
| `exec_sum` | 全局正在执行剧本数 |
| `{uuid}&&exec_sum` | 某剧本正在执行数 |
| `api_key` | 系统 API Key 缓存 |
| `timer_status:{timer_uuid}` | 定时任务状态缓存 |

## 13. 迁移约束

- P0 表结构变更通过 `migrations/mysql/000002_p0_tables.sql` 新增。
- 新增字段通过 `ALTER TABLE` 迁移，不直接修改线上表。
- 所有表必须包含 `create_time` 和 `update_time`。
- 配置类表必须包含 `deleted_at` 软删除字段。
