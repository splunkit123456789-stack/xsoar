# SOAR 平台 产品需求文档

## 1. 产品定位

**SOAR 平台** 是一款面向企业安全运营与运维自动化的**低代码编排平台**（Security Orchestration, Automation and Response）。

核心价值：把不同的系统、组件通过 **APP** 进行封装形成平台能力，通过**剧本**画出逻辑过程，利用多种 **Trigger** 实现自动化执行。

适应场景：DevOps、安全运营、自动化渗透、工作流程自动化等。

## 4. 阶段目标概览

| 阶段 | 目标 | 关键能力 |
|---|---|---|
| MVP | 打通"剧本编辑→执行→日志"核心闭环 | 登录、APP 加载、图形化剧本编辑器、手动执行、执行日志、基础 APP 集合 |
| P0 | 生产可用底线 | 定时触发器、Webhook 触发器、用户角色管理、剧本分类、全局变量、仪表盘、APP 分组、执行结果查询、CORS、WebSocket 实时推送 |
| P1 | 重要增强 | 剧本审批流、审计日志、条件分支节点、循环节点、并行执行、版本管理、剧本导入/导出、APP 市场/导入、节点输出变量提取、执行历史与重跑、双主题 UI、APP 参数类型完善、错误处理节点、告警通道 |
| P2 | 高级特性扩展 | 分布式执行引擎、多租户/组织空间、RBAC 细粒度权限、SSO 单点登录、剧本执行报告导出、大屏可视化、APP 开发 SDK 与脚手架、剧本调试模式、资产/CMDB 集成、SIEM 集成与告警双向同步、性能监控与限流、备份与恢复、API 开放平台、国际化 i18n |
| P3 | 生态与前瞻能力 | AI 智能剧本生成、AI 异常根因分析、APP 商店、低代码可视化大屏搭建、ChatOps 集成、剧本模板库、Kubernetes Operator 化、边缘轻量化部署、区块链审计存证、可观测性集成、多云混合编排 |

## 5. 非目标

- 不承诺兼容 Splunk SOAR / Demisto / Phantom 等商业 SOAR 平台的剧本格式。
- 不提供完整 SIEM 告警分析能力（SOAR 是 SOAR 平台，不是 SIEM）。
- MVP/P0 阶段不引入多租户、完整 RBAC、审批流、AI 能力。
- 不提供完整 DevOps 流水线（Jenkins/GitLab CI 替代品）。

## 6. 技术栈基线

| 维度 | 选型 | 说明 |
|---|---|---|
| 后端语言 | Python 3.9+ | 现状为 Flask 1.1.4，需升级 |
| Web 框架 | Flask 2.x+ | 升级依赖 |
| ORM | Flask-Orator | 现状；P1 评估迁移 SQLAlchemy |
| 数据库 | MySQL 8.0+ | 主存储 |
| 缓存 | Redis 6+ | 会话/缓存/调度状态 |
| 任务调度 | APScheduler (gevent) | 现状；P2 评估 Celery |
| 分布式执行 | RPyC | 现状 |
| 实时通信 | WebSocket (Flask-Sockets + gevent) | 现状 |
| 前端 | Vue3 + Vite + Element Plus | 参照 xdp 前端规范 |
| 部署 | Docker / Docker-Compose / 源码 | 现状 |

## 7. 拆分原则

参照 `xdp` 项目文档规范：

- **需求设计**：描述用户目标、页面能力、功能边界、优先级和不做什么。
- **API 设计**：描述接口路径、请求响应、鉴权、错误码和阶段稳定性。
- **数据库设计**：描述 MySQL 表结构、Redis 使用、迁移约束。
- **人工验收**：描述浏览器操作、命令行造数、验证步骤和通过标准。
